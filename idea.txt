1) Для асинхронного воспроизведения аудио нужна библиотека from play_sount import play_file_async.
Она позволяет запустить файл и продолжить воспроизведение кода, что позволит
спамить звуками
пример структуры
import asyncio
from play_sounds import play_file_async

async def main():
    # Воспроизведение не заблокирует выполнение
    await play_file_async("ваш_файл.mp3", block=False)
    print("Звук играет в фоне, а код выполняется дальше!")
    await asyncio.sleep(5) # Имитация другой работы

asyncio.run(main())
2) Есть вариант использолвать pygame и несколько потоков:
import pygame
import threading
import time

def play_music(file_path):
    pygame.mixer.init()
    pygame.mixer.music.load(file_path)
    pygame.mixer.music.play()
    # Имитация ожидания окончания трека в потоке
    while pygame.mixer.music.get_busy():
        time.sleep(1)

# Запуск воспроизведения в фоновом потоке
music_thread = threading.Thread(target=play_music, args=("ваш_файл.mp3",))
music_thread.start()

print("Музыка играет в фоне, а программа работает дальше!")

3) Так же должна быть возможность конвертации аудиофайлы в mp3,
что бы не ебстись с чтением файлов другмх форматов. Это необходимо для
добавления собственных аудиофайлов. Для этого можно использовать
pydub + ffmpeg: 
pydub — это высокоуровневая библиотека, которая делает конвертацию
очень простой, а ffmpeg выступает в роли мощного бэкенда для работы 
с огромным количеством форматов
Пример применения:
from pydub import AudioSegment

# Загрузка аудиофайла (например, m4a)
audio = AudioSegment.from_file("исходный_файл.m4a", format="m4a")
# Или для mp3, wav и т.д.:
# audio = AudioSegment.from_file("исходный_файл.mp3")

# Экспорт в MP3 с нужным битрейтом
audio.export("конечный_файл.mp3", format="mp3", bitrate="192k")
print("Конвертация завершена!")
